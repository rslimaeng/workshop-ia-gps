P1 — Redigir Memorial Descritivo

⚙️ Engenharia 
Papel: Você é um redator técnico especialista em documentação de engenharia civil.

Contexto: Estou preparando um memorial descritivo para um projeto residencial unifamiliar
em Fortaleza/CE. O projeto é [tipo: elétrico / hidrossanitário / estrutural].
As normas aplicáveis são [ex: NBR 5410 para instalações elétricas].

Tarefa: Redija o memorial descritivo com as seguintes informações:
- Nome do projeto: [nome]
- Área construída: [m²]
- Padrão construtivo: [popular / médio / alto]
- Especificações principais: [liste os itens principais, ex: "fiação 2,5mm², eletrodutos PVC rígido, quadro de distribuição da marca X"]
- Responsável técnico: [nome do engenheiro]

Formato: Documento técnico formal. Use subtítulos por sistema (fundações, estrutura, instalações, acabamentos conforme aplicável). Linguagem técnica, mas clara. Máximo 2 páginas.

Limitações: Não invente especificações que não foram informadas. Se faltar alguma informação essencial, me pergunte antes de redigir.

P2 — Estruturar Ata de Reunião de Obra

⚙️ Engenharia 
Papel: Você é um assistente técnico especializado em documentação de obras.

Contexto: Realizei uma reunião de obra na data [data] com os seguintes participantes:
[nomes e funções]. A obra fica em [endereço ou nome do projeto].

Tarefa: Com base nas minhas anotações abaixo, estruture uma ata de reunião de obra
com: data, participantes, assuntos tratados, pendências identificadas, decisões tomadas,
responsáveis por cada ação e prazo.

Minhas anotações brutas:
[cole aqui suas anotações da reunião, mesmo que bagunçadas]

Formato:
- Cabeçalho com data, obra, participantes
- Seção "Assuntos tratados" (em tópicos)
- Tabela de pendências: Pendência | Responsável | Prazo | Status
- Seção "Próxima reunião" (data prevista se houver)
Tom: formal e objetivo, sem texto desnecessário.

Limitações: Use apenas as informações que eu forneci. Não adicione pendências ou decisões que não estejam nas anotações.

P3 — Interpretar Trecho de Norma Técnica

⚙️ Engenharia 
Papel: Você é um consultor técnico especializado em normas brasileiras de construção civil.

Contexto: Sou engenheiro/técnico da GPS Engenharia e preciso entender um trecho
de norma para aplicar em projeto específico.

Tarefa: Explique em linguagem prática o que significa o trecho de norma abaixo,
usando um exemplo real de aplicação em obra residencial ou comercial:

[cole aqui o trecho da norma]

Norma: [ex: NBR 6118, NR-10, SINAPI tabela X]

Formato:
1. Resumo do que a norma exige (2-3 frases simples)
2. Exemplo prático de como aplicar isso em obra
3. O que NÃO fazer (armadilhas comuns)

Limitações: Se o trecho for ambíguo, me diga. Não interprete além do que está escrito.
Para questões que exijam decisão técnica com consequências de segurança ou responsabilidade,
sempre oriente a consultar um especialista ou o próprio órgão normativo.

P4 — Revisar Orçamento / Verificar Itens Faltantes

⚙️ Engenharia 
Papel: Você é um especialista em orçamentos de construção civil com experiência em SINAPI.

Contexto: Estou elaborando um orçamento para [tipo de obra: residencial / comercial / reforma].
A obra está localizada em Fortaleza/CE. Já tenho os seguintes itens na planilha:

[liste ou cole os itens que você já tem]

Tarefa:
1. Analise se os itens estão coerentes entre si
2. Aponte possíveis itens faltantes típicos para esse tipo de obra
3. Identifique qualquer inconsistência ou item que parece mal dimensionado

Formato:
- Lista de verificação: ✓ OK / ⚠️ Verificar / ✗ Faltando
- Breve justificativa para cada apontamento
- Sugestão dos itens mais críticos que precisam ser conferidos

Limitações: Você não tem acesso a quantitativos detalhados. Baseie-se nos itens que eu forneci.
Não calcule preços — apenas sinalize categorias e coerência.

P5 — Criar Especificação Técnica de Material

⚙️ Engenharia 
Papel: Você é um engenheiro especialista em especificações técnicas para construção civil.

Contexto: Preciso especificar [tipo de material, ex: "cabos elétricos para instalação embutida"]
para um projeto [tipo do projeto] localizado em [ambiente: interno / externo / molhado / seco].

Tarefa: Crie uma especificação técnica completa para este material, incluindo:
- Descrição do material
- Norma de referência (ABNT NBR correspondente)
- Características mínimas exigidas
- Condições de instalação
- Critérios de aceitação e rejeição
- Marcas referência (se aplicável)

Formato: Documento técnico. Tabela para características. Tom formal de projeto.

Limitações: Baseie-se nas normas ABNT vigentes. Se não souber a norma específica, diga e sugira onde consultar.

P6 — Criar Relatório Executivo para Diretoria

💼 Administrativo 
Papel: Você é um analista de gestão especializado em relatórios executivos para PMEs.

Contexto: Sou [função] da GPS Engenharia e preciso apresentar [tema: resultado do mês,
status de projeto, análise de fornecedores] para a diretoria.

Tarefa: Com base nos dados abaixo, estruture um relatório executivo de 1 página:

[cole os dados brutos — números, situações, problemas]

Formato:
- Título e data
- Resumo executivo (3-4 linhas: o que é, qual o resultado, o que está em risco)
- Tabela ou gráfico textual dos números principais
- Pontos de atenção (máximo 3)
- Recomendação ou próximos passos

Tom: direto, sem rodeios. Quem vai ler não tem tempo para texto longo.

Limitações: Use apenas os dados que eu fornecer. Se faltar dado essencial para o relatório ficar completo, me avise.

P7 — Estruturar Pauta de Reunião

💼 Administrativo 
Papel: Você é um facilitador de reuniões corporativas.

Contexto: Vou conduzir uma reunião de [tipo: alinhamento de projeto / revisão de resultados /
planejamento] com [número] pessoas, duração prevista de [tempo].

Os objetivos da reunião são:
[liste 2-3 objetivos]

Tarefa: Crie uma pauta estruturada com tempo alocado por item, responsável por conduzir
cada ponto, e a pergunta-gatilho para abrir cada assunto.

Formato: Tabela com colunas: Horário | Item | Responsável | Objetivo do item | Tempo
Adicione: "Preparação para quem vai participar" — o que cada pessoa deveria ler ou trazer.

Limitações: Não exceda o tempo total de [X] minutos. Deixe sempre 5-10 min para dúvidas e encerramento.

P8 — Analisar Proposta de Fornecedor

💼 Administrativo 
Papel: Você é um especialista em análise de fornecedores e contratos para construção civil.

Contexto: Recebi uma proposta comercial de [fornecedor / tipo de serviço] para [finalidade].
O valor total da proposta é [R$].

Tarefa: Analise a proposta abaixo e me diga:
1. Quais itens estão claramente descritos e quais estão vagos
2. O que está faltando e deveria estar na proposta
3. Quais cláusulas ou condições merecem atenção especial
4. 3 perguntas que devo fazer ao fornecedor antes de fechar

[cole o texto da proposta]

Formato: Use os 4 blocos acima. Seja direto — 2-3 linhas por item, sem texto decorativo.

Limitações: Não invente informações. Se a proposta não mencionar algo, registre como "não informado".

P9 — Redigir Comunicado Interno

👥 RH / DP 
Papel: Você é um especialista em comunicação interna corporativa.

Contexto: Preciso comunicar [assunto] para os colaboradores da GPS Engenharia.
O público é [todos / equipe técnica / setor administrativo].
O tom deve ser [formal / amigável / urgente].

Informações a incluir:
[liste os pontos que precisam estar na comunicação]

Tarefa: Redija um comunicado interno claro, objetivo e com linguagem adequada ao público.

Formato:
- Assunto (linha de e-mail)
- Corpo do comunicado (máximo 15 linhas)
- Fechamento com nome/cargo do remetente

Limitações: Não adicione informações que não estejam nos pontos que listei.

P10 — Resumir Documento Trabalhista

👥 RH / DP 
Papel: Você é um especialista em legislação trabalhista brasileira e comunicação acessível.

Contexto: Preciso entender um trecho de [documento: CLT / acordo coletivo / política interna / NR]
para aplicar no contexto da GPS Engenharia.

Tarefa: Explique em linguagem simples o que este trecho determina, usando exemplos práticos
do dia a dia de uma empresa de engenharia:

[cole o trecho do documento]

Formato:
1. O que a lei/norma diz (resumo em 3 linhas)
2. O que isso significa na prática (exemplo concreto)
3. O que a empresa precisa fazer para estar em conformidade

Limitações: Se houver ambiguidade legal, diga que é necessário consultar um advogado trabalhista.
Não dê opinião jurídica — explique o que está escrito.

P11 — Aprender uma Habilidade Nova Mais Rápido

🎓 Estagiários 
Papel: Você é um tutor especializado em aprendizado de habilidades técnicas.

Contexto: Sou [função: estagiário de engenharia / jovem aprendiz] e preciso aprender
[habilidade: como usar o AutoCAD para plantas baixas / como calcular quantitativos /
como ler uma tabela SINAPI]. Tenho [tempo disponível: 30 minutos / 1 hora / 1 semana].

Tarefa: Crie um plano de aprendizagem prático para eu dominar o básico essencial
desta habilidade no tempo que tenho.

Formato:
- O que é o essencial para fazer o trabalho (20% que resolve 80%)
- Sequência de aprendizagem (lista de etapas)
- 2-3 recursos gratuitos para cada etapa
- 1 exercício prático para testar o que aprendi

Limitações: Seja realista sobre o que dá para aprender no tempo que tenho.
Não crie uma lista enorme que não cabe no prazo.

P12 — Revisar Texto / E-mail Antes de Enviar

🎓 Estagiários 
Papel: Você é um revisor de comunicação profissional.

Contexto: Preciso enviar este [e-mail / mensagem / relatório] para [destinatário: gestor / cliente / fornecedor].
Quero que fique profissional e claro.

Tarefa:
1. Corrija erros de português
2. Melhore a clareza sem mudar o que estou querendo dizer
3. Ajuste o tom para que seja [mais formal / mais direto / mais gentil]
4. Indique se tem alguma informação que deveria estar e não está

Texto original:
[cole aqui seu texto]

Formato: Me dê a versão revisada + uma lista rápida do que você mudou e por quê.

Limitações: Não mude o sentido do que estou dizendo. Se não entender a intenção de algum trecho, me pergunte.

Dicas de uso

Prompt Ficou Ruim? Ajuste Estes Pontos

Problema O que ajustar
Resposta muito genérica Adicione mais contexto sobre a situação específica
Tarefa não foi bem executada Seja mais específico no que quer como saída
Formato errado Especifique: tabela, lista, número de parágrafos, etc.
IA inventou dados Revise sempre normas, preços SINAPI e dados técnicos
Resposta muito longa Adicione "máximo X linhas" ou "de forma concisa" nas Limitações
Protocolo anti-alucinação para documentos técnicos: sempre que usar IA para normas, preços SINAPI ou especificações de segurança: (1) peça à IA para indicar a norma, (2) confirme manualmente o número, (3) verifique se está vigente, (4) só então use no documento assinado.
