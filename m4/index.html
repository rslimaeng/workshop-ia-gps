O problema

Um Prompt Ruim é Sempre Culpa do Humano

A IA é previsível: ela gera saídas proporcionais à qualidade da entrada. Um prompt vago gera resposta vaga. Um prompt preciso gera resposta precisa. A engenharia de prompt é a habilidade de escrever a entrada certa.

Prompt Problema Resultado provável
"Faz um memorial descritivo" Sem contexto, sem especificações, sem formato Genérico, inútil, precisa refazer tudo
"Melhora esse texto" Sem critério de qualidade, sem destinatário Versão levemente diferente do original
"Explica a NBR 5410" Norma extensa, sem recorte de aplicação Resumo genérico de 10 parágrafos

O método

PCTFL — Os 5 Elementos de um Prompt Profissional

Letra Elemento O que define Exemplo (engenharia GPS)
P Papel Quem a IA deve ser "Você é um redator técnico especialista em documentação de engenharia civil"
C Contexto A situação real "Estou preparando um memorial descritivo para projeto elétrico residencial em Fortaleza/CE. Norma: NBR 5410."
T Tarefa O que fazer "Redija o memorial com: área 120m², padrão médio, fiação 2,5mm², eletrodutos PVC rígido."
F Formato Como entregar "Documento técnico formal. Subtítulos por sistema. Máximo 2 páginas."
L Limitações O que não fazer "Não invente especificações. Se faltar informação, me pergunte antes de redigir."
Regra de ouro: quanto mais você preenche o PCTFL, menos a IA precisa adivinhar — e menos chance de alucinação ou resultado genérico.

Exemplo completo

Prompt Real: Memorial Descritivo Elétrico

Este é exatamente o Prompt 1 do Banco de Prompts GPS — com o PCTFL desmontado para você ver cada parte:

PAPEL

"Você é um redator técnico especialista em documentação de engenharia civil."

CONTEXTO

"Estou preparando um memorial descritivo para um projeto elétrico residencial unifamiliar em Fortaleza/CE. A norma aplicável é NBR 5410."

TAREFA

"Redija o memorial descritivo com: nome 'Residência Família Silva', área 120m², padrão médio, fiação 2,5mm², eletrodutos PVC rígido, quadro de distribuição da marca Schneider. Responsável técnico: Eng. João Araújo."

FORMATO

"Documento técnico formal. Use subtítulos por sistema. Linguagem técnica, mas clara. Máximo 2 páginas."

LIMITAÇÕES

"Não invente especificações que não foram informadas. Se faltar alguma informação essencial, me pergunte antes de redigir."

EXERCÍCIO

Monte um Prompt PCTFL para sua Função

Escolha uma tarefa real do seu trabalho que você fez na última semana. Monte o prompt nos 5 elementos:

Elemento Sua resposta
PapelVocê é um ____
ContextoEstou em ____, o projeto é ____, a situação é ____
TarefaFaça ____ com os seguintes dados: ____
FormatoEntregue como ____ (tabela / lista / documento / e-mail)
LimitaçõesNão ____ · Se faltar ____, me pergunte
Depois de escrever: teste no ChatGPT ou Claude. Compare o resultado com o que você obteria com um prompt simples. A diferença é o valor do método.

Critério de sucesso: o resultado precisa de menos de 5 minutos de revisão para estar pronto pra usar. Se demorar mais, o prompt precisa de mais contexto.

Do individual para o coletivo

Padrões Internos de Uso

Um prompt que funciona é ativo da empresa — não só do engenheiro que criou. A diferença entre "tenho um prompt bom" e "a GPS tem um padrão de prompt" é uma página de documentação.

Template para documentar um prompt

Campo O que preencher
Nome do prompt Ex: "Memorial Descritivo — Obras Urbanas"
Tipo de tarefa Ex: Geração de documento técnico
Prompt PCTFL completo Cole o prompt que funcionou — exatamente como você escreveu
Resultado esperado Ex: Memorial de 1-2 páginas, tom formal, subtítulos por sistema
Revisão necessária Ex: Verificar especificações técnicas, ajustar dados reais do projeto
Quem criou / quando Responsabilidade e rastreabilidade — sem isso, prompt vira lenda urbana
Onde guardar: pasta compartilhada da empresa (Google Drive, OneDrive), Notion da GPS, ou um Google Docs com os prompts do time. O que importa é estar acessível para quem precisar — não na cabeça ou no chat individual.

Bônus · Próximo nível

🚀 Do Prompt ao Assistente Persistente

Você criou um prompt PCTFL. O próximo passo é transformá-lo em um assistente persistente — um parceiro especializado que já sabe quem você é e o que você precisa, sem você re-explicar a cada sessão nova.

Isso é o salto concreto para os níveis mais altos dos 7 Níveis de Uso de IA que vimos no M1 — sair do prompt isolado e operar com contexto persistente.

Caminho 1

No Claude (gratuito)

  1. Abra Claude → menu lateral → Projects → New Project
  2. Em Project Instructions, cole o contexto da GPS e o padrão que você criou neste M4
  3. Teste numa nova conversa: a IA já sabe quem você é, sem reexplicar

"Sou engenheiro na GPS Engenharia, Fortaleza/CE. Trabalhamos com projetos residenciais, comerciais e licitações municipais. Tom padrão: técnico e objetivo."

Caminho 2

No ChatGPT (gratuito)

  1. Menu → Personalizar ChatGPT → Instruções personalizadas
  2. Em "O que devo saber sobre você": cole o contexto da sua função na GPS
  3. Em "Como deve responder": cole o padrão de formato que funcionou neste M4

Pronto. Toda conversa nova já começa com o ChatGPT sabendo seu papel, seu contexto técnico e o tom que você espera.

O que muda na prática: você não re-explica mais quem é e o que faz. A IA já sabe. É como ter um colega que está sempre em dia com seus projetos — sem onboarding a cada sessão nova.

Conexão com os 7 Níveis: isso é o salto do N4 (Frequente Profissional) para o N5 (Integrador) — você passa a operar com assistentes especializados, não com prompts isolados. Quando você compartilhar esse assistente com o time, entra no território do N6 (Construtor).
